# 72 ÔÇö Continuous Integration

**Owns:** the GitHub Actions pipeline ÔÇö triggers, permissions, concurrency, environment, every job and step, caching, and the system dependencies it installs.

**Sources:** `.github/workflows/ci.yml`, `mise.toml` (task definitions the steps invoke ÔÇö owned by [71-toolchain](71-toolchain.md)).

---

## 1. Workflow: `CI` Ôťů

The repository has exactly one workflow, `.github/workflows/ci.yml`. There is no
release workflow ([73-release](73-release.md)), no matrix, and a single job.

### 1.1 Triggers

| Event | Filter |
|---|---|
| `push` | branches: `master` only |
| `pull_request` | all PRs (no filter) |

### 1.2 Workflow-level settings

- **Permissions:** `contents: read` (least-privilege token; nothing writes back).
- **Concurrency:** group `ci-${{ github.ref }}` with `cancel-in-progress: true` ÔÇö a
  new push to the same ref cancels the running build.
- **Env:** `CARGO_TERM_COLOR: always`, `RUST_BACKTRACE: "1"`.

## 2. The `ci` job

Runs on `ubuntu-latest`. No OS/toolchain matrix ÔÇö Linux x86_64 with the pinned
toolchain is the only CI-covered configuration (see Divergences). Steps in order:

| # | Step | Action / command | Notes |
|---|---|---|---|
| 1 | Checkout | `actions/checkout@v4` | |
| 2 | Native build deps | `sudo apt-get update && sudo apt-get install -y cmake clang libclang-dev libssl-dev pkg-config` | Per the step name: `cmake` + C++ for usearch, `cc` for rusqlite (bundled SQLite) and tree-sitter, `libssl-dev`/`pkg-config` for ort. `clang`/`libclang-dev` for bindgen-style builds. |
| 3 | Rust toolchain | `dtolnay/rust-toolchain@stable` with `toolchain: "1.96.0"`, `components: rustfmt, clippy` | Hardcodes the version a second time ÔÇö the same value as `rust-toolchain.toml` but not read from it (see Divergences). |
| 4 | Dev tools | `jdx/mise-action@v2` with `install: true` | Installs everything in `mise.toml [tools]`: taplo, typos, actionlint, pkl, hk, cargo-nextest, cargo-deny. |
| 5 | Cache | `Swatinem/rust-cache@v2` | Default configuration; caches `~/.cargo` registry/git and `target/` keyed on lockfile + toolchain. The only cache in the pipeline (mise tools are re-resolved each run unless mise-action's own caching applies). |
| 6 | Format check | `mise run fmt-check` | `cargo fmt --all --check` + `taplo fmt --check`. |
| 7 | Clippy | `mise run clippy` | `--workspace --all-targets --all-features -- -D warnings` ÔÇö the one step that compiles the optional `pdfium` feature. |
| 8 | Build | `mise run build` | `cargo build --workspace --all-targets` (default features). |
| 9 | Test | `mise run test` | `cargo nextest run --workspace` (default features). |
| 10 | Supply-chain & spelling | `mise run lint` | `typos` + `actionlint` + `cargo deny check` ([71-toolchain](71-toolchain.md) ┬ž6). |

All quality steps delegate to mise tasks so local and CI invocations share
definitions ([71-toolchain](71-toolchain.md) ┬ž3.2) ÔÇö but the pipeline calls five
tasks individually rather than the aggregate `ci` task, because the local `ci` task
lacks `build`.

## 3. Relationship to local gates

Three gates run overlapping subsets of the same tasks:

| Gate | fmt-check | typos/actionlint/deny | clippy `-D warnings` | build | test |
|---|---|---|---|---|---|
| CI (this doc) | Ôťů | Ôťů | Ôťů | Ôťů | Ôťů |
| `mise run ci` | Ôťů | Ôťů | Ôťů | ÔÇö | Ôťů |
| hk `pre-push` | fmt via linters | typos/actionlint (no deny) | Ôťů | ÔÇö | Ôťů |
| hk `pre-commit` | fixes instead of checks | typos/actionlint (no deny) | ÔÇö | ÔÇö | ÔÇö |

## Divergences & open questions

- **Toolchain version is duplicated.** `ci.yml` hardcodes `"1.96.0"` in the
  `dtolnay/rust-toolchain` step instead of reading `rust-toolchain.toml`; a toolchain
  bump requires editing two files, and nothing detects drift between them.
- **No MSRV job.** `rust-version = 1.88` ([71-toolchain](71-toolchain.md) ┬ž8.1) is
  never built in CI, so the declared floor is unverified.
- **No platform matrix.** The product targets Linux, macOS, and FreeBSD
  (CONTRIBUTING.md, PRD ┬ž8) and the release plan targets `{x86_64,aarch64}-{linux,macos}`
  ([73-release](73-release.md)), but CI builds only `ubuntu-latest` x86_64.
- **Feature coverage is asymmetric.** Only clippy runs `--all-features`; `build` and
  `test` run default features, so the `pdfium` feature is lint-compiled but never
  built into or exercised by tests in CI.
- **hk and pkl are installed in CI but unused.** mise-action installs the whole
  `[tools]` set, including the hook manager and the Pkl CLI, yet no step runs
  `hk check` or validates `hk.pkl` ÔÇö CI cannot catch a broken hook config.
- **Actions are tag-pinned, not SHA-pinned** (`@v4`, `@v2`, `@stable`). Given the
  supply-chain posture in SECURITY.md and deny.toml, mutable tags are the weakest
  link in the pipeline.
- **`mise run ci` is advertised as mirroring CI** (CONTRIBUTING.md, README.md) but
  omits the `build` step; a workspace that clippy-passes yet fails `--all-targets`
  codegen would only surface in GitHub CI.
